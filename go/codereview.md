# Советы кейсы на что обращать внимание при код ревью

# Проектирование  API
- если будет какой нибудь глосаррий, то идеально, чтобы одно и тоже не называли по разному в разных ручках
- лимиты
  - есть в реквесте есть массив, то должно быть ограничение сверху (чтобы вам не синули милион/милиард)
  - если ручка отдает список, то тоже должен быть максимальный предел (не больше 1000 записей за запрос)
  - пагинация, если ручка отдает список, нужно закладывать пагинация, не всегда все поместиться в один ответ
 
# Grpc/HTTP вызовы сервисов
- проставляем/подбираем таймауты
- если получаем список, то нужно понять с каким лимитом можно ходить.

# Многопоточность/конкаренси/горутины
- если есть горутина, то нужно не забывать про мьютексы
- если работаем с каналами, то не забываем закрывать
