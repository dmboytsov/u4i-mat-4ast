# Saga
Патерн для распределенной транзакции.  **Проблема** в микросервисной архитектуре нету транзакций как на бд, у каждого своя база.
**Идея**. У каждого сервиса своя транзакция, она зависит от других транзакций в рамках одной операции, если какое-то шаг не получается выполнить запускаются компенсирующие действия. Получается что система **Eventual Consistency**, не консистентна в моменте, но в итоге должна прейти в консистентное состояние.
